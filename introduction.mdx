# MS200 LiDAR Driver for ROS 2

A lightweight, native ROS 2 driver for the **MKS MS200 / Oradar MS200** Time-of-Flight LiDAR sensor. Written entirely in Python, this driver implements the low-level serial communication protocol to decode sensor data packets and publish standard `sensor_msgs/LaserScan` messages — with no proprietary SDKs required.

## What is the MS200?

The MKS MS200 (also marketed as the Oradar MS200) is a compact, affordable 360-degree TOF LiDAR sensor with a range of up to 12 meters. It communicates over a serial (UART) interface, making it easy to interface with single-board computers like the Raspberry Pi.

## Key features

## Hardware requirements

| Component | Details |
|-----------|---------|
| **Sensor** | MKS MS200 / Oradar MS200 TOF LiDAR |
| **Interface** | USB-to-Serial adapter (e.g., CP2102, CH340) |
| **Platform** | Linux-based system (Raspberry Pi, x86_64) with ROS 2 installed |
| **ROS 2 version** | Tested with ROS 2 Jazzy |

## Software dependencies

The driver depends on standard ROS 2 packages and the Python `pyserial` library:

- `rclpy` — ROS 2 Python client library
- `sensor_msgs` — Standard sensor message types
- `std_msgs` — Standard message types
- `python3-serial` — Python serial port communication

<Info>
  This driver has been tested on a **Raspberry Pi 5** running **ROS 2 Jazzy**. It should work on any Linux-based platform with ROS 2 installed.
</Info>
