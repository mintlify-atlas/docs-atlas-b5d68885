# Quickstart

This guide walks you through cloning, building, and running the MS200 LiDAR driver in a ROS 2 workspace.

## Prerequisites

- ROS 2 installed (Jazzy recommended)
- A ROS 2 workspace (e.g., `~/ros2_ws`)
- MKS MS200 / Oradar MS200 LiDAR connected via USB-to-Serial adapter
- `rosdep` available for dependency installation

## Steps

<Steps>
  <Step title="Clone the repository">
    Navigate to your ROS 2 workspace source directory and clone the driver:

    ```bash
    cd ~/ros2_ws/src
    git clone https://github.com/jasonmm24/ms200_driver.git
    ```
  </Step>

  <Step title="Install dependencies">
    From the workspace root, use `rosdep` to install all required dependencies:

    ```bash
    cd ~/ros2_ws
    rosdep install --from-paths src --ignore-src -r -y
    ```

    This installs `python3-serial` and any other declared dependencies from `package.xml`.
  </Step>

  <Step title="Build the package">
    Build the `ms200_driver` package using `colcon`:

    ```bash
    colcon build --symlink-install --packages-select ms200_driver
    source install/setup.bash
    ```

    <Tip>
      Using `--symlink-install` means Python source changes are reflected immediately without rebuilding.
    </Tip>
  </Step>

  <Step title="Grant serial port access">
    Ensure your user has permission to access the USB serial port:

    ```bash
    sudo chmod 777 /dev/ttyUSB0
    ```

    For permanent access, add your user to the `dialout` group (requires logout/reboot):

    ```bash
    sudo usermod -a -G dialout $USER
    ```
  </Step>

  <Step title="Run the driver">
    Start the MS200 node with default settings (`/dev/ttyUSB0`, 230400 baud):

    ```bash
    ros2 run ms200_driver ms200_node
    ```

    You should see output confirming the connection:

    ```
    [INFO] [ms200_node]: Conectado al LiDAR en /dev/ttyUSB0 a 230400
    ```
  </Step>

  <Step title="Verify data output">
    In a second terminal, verify the driver is publishing scan data:

    ```bash
    ros2 topic echo /scan --once
    ```

    You should see a `sensor_msgs/msg/LaserScan` message with populated `ranges` data.
  </Step>
</Steps>
